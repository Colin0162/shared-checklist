목차 ① 이 앱은 ② 기능 ③ 만들어 온 순서 ④ 기술 ⑤ 직접 수정하기 ⑥ 실행·배포

① 이 앱은

여러 명이 링크 하나로 접속해 함께 체크하는 실시간 공유 체크리스트예요. MT·교리·행사 준비물부터 일정표·답사 평가까지, 폴더로 정리하고 다 같이 실시간으로 체크하고 메모할 수 있어요. 공유 폴더에서는 채팅도 됩니다.

② 기능

📁 폴더 (정리함)

  • 종류 3가지: 🔒 개인 나만 / 🤝 공유 암호 아는 사람 / 📁 공개 누구나
  • 새 폴더는 무조건 개인 → 만든 뒤 ‘공유’ 버튼으로 암호(키워드)를 정해 공유로 전환(최상위 폴더만)
  • 다른 사람은 ‘🔑 공유 폴더 참여’에 암호를 넣으면 그때부터 폴더가 보임
  • 폴더·게시글 이동(한 단계씩 들어가며 선택) / 삭제(빈 폴더, 재확인)

📝 게시글 (체크리스트)

  • 형식 4가지: 체크박스 / 상·중·하 평가 / 할 일 / 표(일정표)
  • 대항목·수량·담당자·비고칸·행사일(D-day)·템플릿 저장/불러오기
  • 입장 비밀번호(추가·변경·삭제) + 편집(관리자) 비밀번호

✅ 함께 쓰기

  • 실시간 공유 체크 + 체크한 사람 표시, 진행률 바, 미완료/담당자 필터
  • 비고 동시 편집 잠금, 인쇄 / PDF 내보내기

💬 공유 폴더 채팅

  • 아이콘(💬)을 누르면 참여자 명단 + 채팅 (컴퓨터=왼쪽 패널 / 폰=드로어)
  • 폴더 관리자: 참여자 내보내기·관리자 넘기기, 공지(📢)
  • 보존: 공지는 영구, 일반 채팅은 최근 24시간 / 삭제는 본인 것만

🔐 계정·모바일

  • 이름+비밀번호 로그인, 가입 승인제, 비밀번호 해시 저장(서버에서 권한 검증)
  • PWA: 홈 화면에 앱처럼 설치 + 오프라인이어도 앱 화면은 열림
  • 폰에서 표는 카드형(라벨 | 값)으로 자동 변환

③ 만들어 온 순서

  1. MVP — 실시간 공유 체크, 이름+PIN 로그인, 체크한 사람 표시
  2. 관리자 빌더 — 게시글 생성/편집, 4형식, 대항목·수량·담당자·비고, 템플릿
  3. 보안 — 비번 해시, 입장/편집 비번, 가입 승인제, 비고 잠금
  4. 안정화 — URL 라우팅, 에러 로깅, 코드 분할, DB 마이그레이션 체계
  5. 폴더 공유 모델 — 개인/공유/공개 + 암호 참여 + 참여자 관리 + 채팅·공지 + 이동/권한
  6. 모바일·마무리 — PWA(설치·오프라인), 폰 표 카드형, 채팅 UX 다듬기

④ 기술

  • 프론트엔드: Vite + React, react-router-dom
  • 백엔드/DB/실시간/인증: Supabase (Postgres + RLS + RPC + Realtime)
  • 호스팅: Vercel · 앱화: vite-plugin-pwa

⑤ 직접 수정하기

규칙 두 가지 — 화면/디자인(src/의 .jsx·.css·.js)은 수정 후 git push하면 자동 배포. DB 구조/규칙은 Supabase의 SQL Editor에서 RUN해야 반영돼요(push만으론 안 됨).

어느 파일이 어느 화면인가

파일역할
src/index.css색 변수(:root) — 전체 톤 바꾸려면 여기
src/App.css거의 모든 디자인(표·채팅·패널 등)
components/Guide.jsx앱 안 ‘사용 가이드’ 문구
components/AppHeader.jsx상단 제목·배너
components/FolderView/FolderList폴더 목록·분류·참여
components/FolderPanel/FolderMembers채팅·참여자 패널
components/AdminEditor.jsx게시글 만들기/편집
supabase/migrations/DB 변경(번호별 파일, 한 번씩 RUN)

색 바꾸기 — src/index.css 맨 위

--accent: #4f46e5;   /* 강조색(버튼·링크) */
--done:   #15a34a;   /* 완료 초록 */
--bg / --card / --text / --border  /* 배경·카드·글씨·테두리 */
자세한 ‘파일 지도·위치표’는 저장소의 EDITING_GUIDE.md에 더 있어요. (이 HTML은 보기 쉽게 추린 요약본)

⑥ 실행·배포

npm run dev      # 로컬 미리보기 (localhost:5173)
npm run lint     # 문법 검사
npm run build    # 빌드 확인
git push         # → Vercel 자동 배포
  • DB 변경은 supabase/migrations/의 새 파일을 SQL Editor에서 RUN
  • .env.local(연결 키)·비밀번호는 깃에 올리지 않기
  • 막히면 npm run build가 통과하는지부터 확인
